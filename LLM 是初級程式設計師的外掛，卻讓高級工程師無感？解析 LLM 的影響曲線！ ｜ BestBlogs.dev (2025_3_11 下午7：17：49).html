LLM 是初級程式設計師的外掛，卻讓高級工程師無感？解析 LLM 的影響曲線！


LLM 到底能不能提高程式設計師的生產力？實際上與程式設計師的經驗水平密切相關。LLM 的影響並非線性增長，而是呈現出一條獨特的曲線：初級工程師受益最大，中高級工程師逐漸感到侷限，而資深工程師又能重新找到它的價值。本文將帶你拆解這條“LLM 影響曲線”，看看不同階段的程式設計師如何看待 LLM 的作用。

關於 LLM 對程式設計師的作用，網上的討論一直很激烈。有些人覺得生產力提升巨大，而有些人則不明白有什麼值得大驚小怪的。每次相關的 Hacker News 討論區裡，總會有一群人爭論不休，我把這稱為“新的鴻溝”。

我有一個理論：LLM 對程式設計師日常工作的影響，主要取決於他們的經驗水平，而且這種影響呈現出一種有趣的曲線。接下來，我會詳細講解這個想法。


這條影響曲線

LLM 在不同等級工程師中的作用並不相同，我把它畫成了一條曲線。如果一眼看不太明白，別急，下面我會逐步解析。


不同等級工程師的影響

初級工程師

作為一名剛入門的初級工程師，在開始接觸一個程式碼庫，很多東西還不熟悉，腦子裡還在建立基本的邏輯框架。這個階段，LLM 簡直是救星！

遇到報錯？LLM 能解釋給你聽。要寫個小功能或者升級庫？LLM 可以幫你更快搞定。

在這個階段，LLM 可能讓你感覺它幾乎能代替你完成大部分工作。但這裡有個潛在的危險：如果你只是把 LLM 當成幫手，像請教資深同事一樣用它，那還好，畢竟現實工作中你也很難迴避它。但如果你只是機械地複製貼上程式碼，而不理解它的邏輯，那你的成長就會受到很大限制。

中級工程師

這個階段，你已經積累了一定的經驗，能夠熟練地在程式碼庫裡遊走。你仍然會發現 LLM 讓你寫程式碼更快，比如用 Copilot 自動補全程式碼、用 AI 代理減少重複程式碼、用 ChatGPT 學習新框架等。

但你也會開始碰到 LLM 解決不了的問題，比如：

  • 它無法幫你理解客戶需求，畢竟它不會分析產品需求單裡的隱藏資訊；

  • 它無法用偵錯工具幫你查出程式碼中的競態問題；

  • 它無法在你凌晨被叫醒處理線上故障時，真正給你有用的幫助。

高級工程師

到了這個階段，你對整個程式碼庫瞭如指掌，甚至有很大一部分是你自己寫的。雖然 LLM 依然能加快你寫程式碼的速度，但問題是——你現在還花多少時間寫程式碼呢？

  • 當你在制定技術規劃時，LLM 幫不上忙。

  • 當你在偵錯一個神秘的“薛定諤的 Bug”時，它會一頭霧水。

  • 當你寫一篇完整的技術設計文件時，它能幫你整理格式和結構，但真正的核心思考，LLM 還是做不到。

LLM 沒有你大腦裡積累的各種上下文資訊，而這些資訊又是無法簡單寫成文字的。這也是為什麼，越是資深的工程師，往往對 LLM 越不感冒——他們的工作內容，LLM 目前還無法替代，尤其是一些涉及深度技術或者特定領域的事情。

資深工程師（Staff+）

這個等級的工程師往往需要探索新方向、為團隊指明道路，而這通常需要大量的實驗。而在這裡，LLM 又開始變得有用起來了。

例如：

  • 你需要驗證某種新技術的可行性，LLM 可以幫你快速搭建一個簡單但能運行的原型；

  • 你想嘗試一種新的架構，LLM 能幫你更快完成初步實現；

  • 你自己腦子裡有足夠的領域知識，所以當 LLM 卡住時，你可以輕鬆幫它“解鎖”。

舉個例子：兩年前，我研究如何把公司大規模程式碼庫中的 RPC 框架遷移到新的解決方案。如果是傳統方式，你可能要先在一個小項目裡測試，再做分析。而借助 LLM，我直接在實際程式碼庫裡做了一版完整的原型。這種“摸得著、跑得動”的方案，比任何設計文件都更有說服力。如今，LLM 的能力只會越來越強，讓資深工程師能夠更快地做實驗、驗證想法。


結論

希望這篇文章能幫大家理解，為什麼有些人對 LLM 超級興奮，而有些人卻完全不感興趣。並不是誰對現實視而不見，而是大家的日常工作內容不同，自然會有不同的感受。

當然，科技發展很快，這條影響曲線可能會在幾年後變成一條水平線，甚至發生完全不同的變化。但有一點可以確定——我們正處在一個激動人心的時代！

摘要

文章深入探討了大型語言模型 (LLM) 對不同經驗水平程式設計師的影響，創新性地提出了 “LLM 影響曲線” 這一概念。該曲線指出 LLM 對初級工程師的幫助最大，可以輔助他們理解程式碼、編寫功能和升級庫；但過度依賴 LLM 可能限制其成長。中級工程師在使用 LLM 時，會發現其無法解決理解客戶需求、偵錯競態問題和處理線上故障等複雜問題。高級工程師則因工作內容側重於技術規劃、複雜問題偵錯和技術文件撰寫，對 LLM 的感知不強。資深工程師則可以利用 LLM 快速驗證新技術、嘗試新架構，加速實驗和想法驗證。文章旨在幫助讀者理解不同程式設計師對 LLM 的不同感受，並探討如何更好地將 LLM 與自身經驗和領域知識相結合，從而更有效地利用 LLM。

主要內容

  • 1. 初級工程師：LLM 是快速入門的助手，但需警惕過度依賴

    初級工程師可借助 LLM 快速理解程式碼和完成任務，但應避免機械複製貼上，而要理解其邏輯，否則會阻礙自身技能的提升。

  • 2. 中高級工程師：LLM 在複雜場景中存在侷限

    LLM 在理解客戶需求、偵錯複雜問題和處理線上故障等需要上下文資訊和領域知識的場景中，無法提供有效幫助。

  • 3. 資深工程師：LLM 可加速原型驗證和創新

    資深工程師可借助 LLM 快速搭建原型、嘗試新架構，從而更高效地探索新方向和為團隊指明道路。
