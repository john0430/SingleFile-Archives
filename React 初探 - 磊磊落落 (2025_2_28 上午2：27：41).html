React 初探

2024年11月21日 電腦

React 是由 Facebook 開發的一個用於建構使用者介面（UI，User Interface）的前端 JavaScript 庫，其專注於檢視（View）層，使開發者能夠更高效地建構單頁應用以及複雜的元件化介面。本文為 React 的初探，首先會介紹 React 的主要特性，然後會以實例的方式介紹其基礎特性的使用。

1 React 有哪些特性？

React 的特性如下：

  • 元件化

    React 將 UI 分解為小的、可重用的元件，每個元件都有自己的狀態和渲染邏輯。元件可以巢狀、組合，實現頁面結構的復用。 可以通過函數式元件和類元件來定義元件，函數式元件更為推薦，因為它們更加簡潔且支援鉤子（Hooks）。

  • 虛擬 DOM

    React 使用虛擬 DOM 來提高性能。它首先在記憶體中建立一個虛擬的 DOM 樹，當元件狀態或資料變化時，React 會計算虛擬 DOM 和實際 DOM 之間的差異，然後進行最小化 DOM 更新。這種方法減少了對實際 DOM 的直接操作，提高了渲染效率，特別是在複雜和頻繁更新的介面中尤為重要。

  • 單向資料流

    React 使用單向資料流，即資料從父元件流向子元件。子元件通過 props 接收資料，不能直接修改父元件的狀態。元件內部的資料有狀態（state）管理，可以通過事件處理函數更新狀態，觸發介面的重新渲染。

  • JSX 語法

    React 使用 JSX（JavaScript XML）來描述 UI 結構。JSX 是一種看起來像 HTML 的語法，它實際是 JavaScript 的語法擴展。JSX 可以讓我們在 JavaScript 程式碼中更直觀地編寫 UI 結構，且具有很好的靈活性和可組合性。

  • Hooks（鉤子）

    React 16.8 引入了鉤子（Hooks），它允許函陣列件使用 state、生命週期等功能，避免了類元件程式碼的冗長。常用的鉤子有 useState、useEffect、useContext、useReducer 等，它們可以用來幫助管理狀態、執行副作用等。

  • Context API

    React 提供了 Context API，其用於在元件樹中共享狀態，而不需要通過 props 層層傳遞。它特別適合全域狀態管理，如使用者認證等。

  • React Router

    React 沒有內建的路由功能，但我們可以使用 React Router 來處理頁面導航。其允許在不刷新頁面的情況下，進行頁面跳轉。

  • 伺服器端渲染（SSR）和靜態生成（SSG）

    React 可以與伺服器端渲染（如 Next.js）結合，提供更好的 SEO 和載入性能。通過 SSR，React 元件在伺服器上預渲染成 HTML，客戶端接管時只需繫結事件，避免了頁面閃爍和延遲。

  • 狀態管理

    React 的狀態管理可以通過元件的 useState 或 useReducer 來完成，複雜應用可以使用外部庫如 Redux、MobX 或 Recoil 來更好地管理全域狀態。

  • 開發者工具

    React 提供了強大的開發者工具，如 React DevTools，可以讓我們偵錯和最佳化元件的渲染、查看元件的樹形結構和狀態等。

總的來說，React 提供了一種靈活、高效且易於維護的方式來建構現代 Web 應用，具有元件化、虛擬 DOM 和強大的生態支援。

2 動手寫一個樣例應用

接下來，我們以實現一個簡單的部落格收集應用程式為例，演示 React 基礎功能的使用。

該部落格應用程式擁有首頁、部落格列表、部落格詳情、部落格提交 4 個頁面。實現後的效果如下：

該應用程式所用到的 Node.js、NPM 和 React 的版本如下：

2.1 範本工程建立

進行編碼前，需要使用如下命令建立出一個僅包含項目骨架的 React 範本工程。

npx create-react-app react-start-demo

骨架工程的目錄結構如下：

│   ├── index.html

├── src/
│   ├── App.js
│   ├── App.test.js
│   ├── index.js
│   └── index.css
└── package.json

接下來，我們會對骨架工程進行一些修改，並基於其之上進行添磚加瓦。

2.3 主要程式碼解析

2.3.1 index.html

public 資料夾下的 index.html 是該 React 工程僅有的一個 html 檔案。其是一個公用範本檔案，定義了 <head> 以及 <body> 中的頭部菜單、底部資訊以及中間待替換部分，該程序的所有頁面均是 React 動態更改該範本頁面的待替換部分（<div class="container" id="root"></div>）來實現的。

<!-- public/index.html -->
<!DOCTYPE html>
<html lang="zh">
  <head>
    <meta charset="utf-8" />
    <title>部落格聚合</title>
  </head>
  <body>
    <header>
      <nav>
        <a href="/">首頁</a>
        <a href="/blogs/add">提交部落格</a>
        <a href="/blogs">部落格列表</a>
      </nav>
    </header>

    <div class="container" id="root"></div>

    <footer>
      <p>© 2024 部落格聚合</p>
    </footer>
  </body>
</html>

2.3.2 index.js

index.js 是該 React 工程的總入口。

// src/index.js
import React from 'react';
import ReactDOM from 'react-dom/client';
import './index.css';
import App from './App';

const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);

可以看到，該檔案引入了 React、ReactDOM 以及一個全域 CSS 檔案 index.css，並使用 ReactDOM 將上述 index.html 檔案中 id="root" 的部分取代為了 <App />。

2.3.3 App.js

App.js 為該應用程式的主檔案，我們在該檔案組態了所有頁面的路由規則。

// src/App.js
import { BrowserRouter as Router, Routes, Route } from 'react-router-dom';
import HomePage from './pages/HomePage';
import BlogAddPage from './pages/BlogAddPage';
import BlogListPage from './pages/BlogListPage';
import BlogDetailPage from './pages/BlogDetailPage';
import NotFoundPage from './pages/NotFoundPage';

export default function App() {
  return (
    <Router>
      <Routes>
        <Route path='/' element={<HomePage />} />
        <Route path='/blogs/add' element={<BlogAddPage />} />
        <Route path='/blogs' element={<BlogListPage />} />
        <Route path='/blogs/:id' element={<BlogDetailPage />} />
        <Route path='*' element={<NotFoundPage />} />
      </Routes>
    </Router>
  );
}

2.3.4 Util 檔案

該應用程式為了簡單，未對接後端 API，其資料儲存使用的是瀏覽器的 localStorage，並使用一個專門的工具類 BlogStorageUtil.js 來提供資料的存取。

// src/utils/BlogStorageUtil.js
export function getAllBlogs() {
    const blogsStr = localStorage.getItem('blogs') || '[]';
    return JSON.parse(blogsStr);
}

export function addBlog(blog) {
    let blogs = getAllBlogs();

    blog.id = blogs.length + 1;
    blogs.push(blog);

    localStorage.setItem('blogs', JSON.stringify(blogs))
}

export function getBlogById(id) {
    return getAllBlogs().find((blog) => blog.id === id);
}

2.3.5 頁面元件

接下來，我們重點介紹一下 src/pages 資料夾下的各個頁面元件。

HomePage.js

HomePage.js 對應該應用程式的首頁，其邏輯非常簡單，僅是更改默認標題，並在範本檔案中的核心區域顯示一段話。

這裡在修改頁面標題時，用到了 React 中的一個 Hook useEffect，主要用於處理副作用（Side Effects）。副作用是指那些不直接影響渲染的操作，比如資料獲取、訂閱事件、手動修改 DOM、定時器等。

// src/pages/HomePage.js
import { useEffect } from 'react';

export default function HomePage() {
    useEffect(() => {
        document.title = '首頁';
    }, []);

    return (
        <p>歡迎訪問部落格聚合，聚合天下優質部落格，讓您在文字的海洋裡徜徉！</p>
    );
}

BlogListPage.js

BlogListPage.js 對應該應用程式的部落格列表頁，該元件除了會動態修改頁面標題外，還會呼叫 BlogStorageUtil 的 getAllBlogs() 方法獲取部落格列表並進行渲染。

// src/pages/BlogListPage.js
import { useEffect } from 'react';
import { getAllBlogs } from '../utils/BlogStorageUtil';

export default function BlogListPage() {
    useEffect(() => {
        document.title = '部落格列表';
    }, []);

    const blogs = getAllBlogs();

    return (
        <div className="blog-list">
            <ul>
                {
                    blogs.map((blog, index) => (
                        <li key={index}>
                            <a href={`/blogs/${blog.id}`}>{blog.name}</a>
                        </li>
                    ))
                }
            </ul>
        </div>
    );
}

BlogDetailPage.js

BlogDetailPage.js 對應該應用程式的部落格詳情頁，該元件除了會動態修改頁面標題外，還會呼叫 BlogStorageUtil 的 getBlogById(id) 方法獲取單個部落格資訊並進行渲染。

// src/pages/BlogDetailPage.js
import { useEffect } from 'react';
import { useParams } from 'react-router-dom';
import { getBlogById } from '../utils/BlogStorageUtil';

export default function BlogDetailPage() {
    let { id } = useParams();

    const blog = getBlogById(Number(id));

    useEffect(() => {
        document.title = blog.name;
    }, [blog]);

    return (
        <div className="blog-detail">
            <h2>{blog.name}</h2>
            <p>{blog.description}</p>
            <div className="note">{blog.technical ? '*該部落格為技術部落格' : '*該部落格為非技術部落格'}</div>
        </div>
    );
}

BlogAddPage.js

BlogAddPage.js 對應該應用程式的部落格新增頁，該元件除了會動態修改頁面標題外，其內有一個 form 表單，會監聽各個欄位的修改。並針對 form 提交，有對應的處理函數。處理函數 handleSubmit() 會對各個欄位的長度進行校驗，處理成功會跳轉到部落格列表頁。

注意，這裡邊除了用到 useEffect Hook 外，還用到一個 useState Hook。useState 是 React 中用於在函陣列件中加入狀態的 Hook。其允許在函陣列件內部聲明狀態變數，並且可以對該狀態變數進行更新。

// src/pages/BlogAddPage.js
import { useEffect, useState } from 'react';
import { addBlog } from '../utils/BlogStorageUtil';

function validateFormData(formData) {
    if (formData.name.length <= 2) {
        return { field: 'name', message: '部落格名稱須大於 2 個字元' };
    }

    if (formData.description.length <= 10) {
        return { field: 'description', message: '部落格描述須大於 10 個字元' };
    }

    return null;
}

export default function BlogAddPage() {
    const [formData, setFormData] = useState({ name: '', description: '', technical: false });
    const [error, setError] = useState({});

    useEffect(() => {
        document.title = '提交部落格';
    }, []);

    const handleChange = (e) => {
        const { name, value } = e.target;
        setFormData({ ...formData, [name]: value });
    };

    const handleSubmit = (e) => {
        e.preventDefault();

        const error = validateFormData(formData);
        if (null === error) {
            addBlog(formData);
            window.location = '/blogs';
        } else {
            setError(error);
        }
    }

    return (
        <div className="form">
            <form onSubmit={handleSubmit}>
                <div>
                    <label>部落格名稱：</label>
                    {error.field === 'name' && <span className="error">{error.message}</span>}
                </div>
                <div>
                    <input name="name" value={formData.name} onChange={handleChange} />
                </div>

                <div>
                    <label>部落格描述：</label>
                    {error.field === 'description' && <span className="error">{error.message}</span>}
                </div>
                <div>
                    <textarea name="description" value={formData.description} onChange={handleChange} />
                </div>

                <div>
                    <label>技術部落格：</label>
                </div>
                <div>
                    <select id="options" name="technical" value={formData.technical} onChange={handleChange}>
                        <option value="false">否</option>
                        <option value="true">是</option>
                    </select>
                </div>

                <div>
                    <button>提交</button>
                </div>
            </form>
        </div>
    );
}
