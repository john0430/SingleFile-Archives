Why My Print Didn't Output Before a Segmentation Fault
為什麼我的列印在分段錯誤之前未輸出

So, as I was learning some C, at the pointers/segfault, I was trying this code myself.
所以，當我學習一些 C 語言時，在 pointers/segfault 處，我自己嘗試了這段代碼。

#include <stdio.h>

int main(void)
{
        printf("%s", "Hello!");
        int *p = NULL;
        *p = 5;
        // Will not be reached due to crash above
        printf("%s", "Another Hello!");
}

Then compile it and run it in the terminal.
然後編譯它並在終端中運行它。

$ gcc -Wall -Wextra -o hello hello.c && ./hello
Segmentation fault (core dumped)

To my surprise, it did not print out both “Hello!” and “Another Hello!” and the program just crashed with “Segmentation fault (core dumped)” (aka segfault).
令我驚訝的是，它沒有列印出 “Hello！” 和 “Another Hello！”，程式只是崩潰了，並顯示 “Segmentation fault （core dumped）”（又名 segfault）。

For “Another Hello!”, I get it. Program crashed before it reached to that line.
對於“Another Hello！”，我明白了。程式在到達該行之前崩潰。

How about “Hello!” ? What’s going on here ? Is it the segfault that prevented printing out ? Before we go further, allow me to expand a bit on this segfault thing.
“Hello！”怎麼樣？這是怎麼回事？是阻止列印出來的段錯誤嗎？在我們進一步討論之前，請允許我稍微擴展一下這個段錯誤的事情。

int *p = NULL;
*p = 5;

The first line declares a pointer p that should hold an integer type and we are setting it to NULL (nowhere/no actual thing).
第一行聲明了一個指標 p，它應該包含一個整數類型，我們將其設置為 NULL（無處/無實際內容）。

In the second line tries to writes “5” to the memory location pointed to by p. But p doesn’t exist. It’s invalid memory access and causes a crash which is segfault!
在第二行中，嘗試將 「5」 寫入 p 指向的記憶體位置。但 p 不存在。這是無效的記憶體訪問，並導致 segfault 崩潰！

Wait, how does this crash prevent the program to print out “Hello!” in terminal? It seems like nothing to do with our printf since it comes before the crash happens.
等等，這個崩潰是如何阻止程式在終端中列印出 「Hello」的呢？這似乎與我們的 printf 無關，因為它發生在崩潰發生之前。

What happens when you do printf ? In most system, it prints out the data to the output device, in this case, the terminal. It turns out the output devices are “line buffered”. Line buffering means the outputs are stored in a place called “buffer” before they are printed out and will be printed out later on certain conditions. The condidtions, according to the GNU C guidelines are
當你做 printf 時會發生什麼？在大多數系統中，它將數據列印到輸出設備，在本例中為終端。事實證明，輸出設備是 「line buffered」 的。行緩衝意味著輸出在列印出來之前存儲在一個稱為“緩衝區”的地方，稍後在某些條件下將被列印出來。根據 GNU C 指南，這些組態是

(Copied straight from the guide)
（直接從指南中複製）

Flushing output on a buffered stream means transmitting all accumulated characters to the file. There are many circumstances when buffered output on a stream is flushed automatically:
刷新緩衝流上的輸出意味著將所有累積的字元傳輸到檔。在許多情況上的緩衝輸出會自動刷新：

  • When you try to do output and the output buffer is full.
    當您嘗試執行 output 並且 output buffer 已滿時。
  • When the stream is closed. See Closing Streams.
    當流關閉時。請參閱 關閉流。
  • When the program terminates by calling exit. See Normal Termination.
    當程式通過調用exit終止時。請參見正常終止。
  • When a newline is written, if the stream is line buffered.
    寫入換行符時，如果流是行緩衝的。
  • Whenever an input operation on any stream actually reads data from its file.
    每當對任何流的輸入作實際從其檔中讀取數據時。

It is true for printing something to the terminal as well as writing something to a file. In our little program, the program crashed due to segfault and did not manage to flush the buffer. The program abruptly ended.
將某些內容列印到終端以及將某些內容寫入檔都是如此。在我們的小程式中，程式由於段錯誤而崩潰，並且沒有設法刷新緩衝區。該計劃突然結束。

Now, how can we force the program to print out “Hello!” in terminal before anything crashed ? That will help with debugging things like until at whihc point my program executed before it crashed.
現在，我們如何強製程式在終端中列印出 「Hello」，以免發生任何崩潰呢？這將有助於調試諸如 until at which point my program perform before it crash 之類的作。

Well, the easy way is remove the error part of the program!
好吧，簡單的方法是刪除程序的錯誤部分！

#include <stdio.h>

int main(void)
{
        printf("%s", "Hello!");
        printf("%s", "Another Hello!");
}

Compile it and run it again!
編譯它並再次運行它！

ubuntu@playground:~$ gcc -Wall -Wextra -o hello hello.c && ./hello
Hello!AnotherHello!ubuntu@playground:~$

Now, all goes well, main() returns successfully and everything is flushed out to the output device. I see my prints!
現在，一切順利，main（） 成功返回，所有內容都已刷新到輸出設備。我看到了我的指紋！

Newline (\n)  換行符 （\n）

But..how about I want to keep things the same way and have my prints. I wanna have the cake and eat it too.
但。。我想保持原樣並擁有我的印刷品怎麼樣。我想吃蛋糕。

Rememer, in one of the conditions above, it says that “When a newline is written, if the stream is line buffered”.
Rememer，在上面的一個條件中，它說 “當寫入換行符時，如果流是行緩衝的”。

For that, I can use a newline character (\n) to force the program flush to the buffer. We can keep the rest in the same way. Let’s see if that works.
為此，我可以使用換行符 （\n） 強製程式刷新到緩衝區。我們可以用同樣的方式保留其餘的。讓我們看看這是否有效。

#include <stdio.h>

int main(void)
{
        // This should prints "Hello!" and
        // flushes to an output device (stdout) due to newline
        printf("%s\n", "Hello!");
        int *p = NULL;
        *p = 5;
        printf("%s\n", "Another Hello!");
}

Yes! I see my “Hello!” here.
是的！我在這裡看到我的 “Hello！”

$ gcc -Wall -Wextra -o hello hello.c && ./hello
Hello!
Segmentation fault (core dumped)

The “Another Hello!” didn’t make it but that’s ok. The program crashed before it reaches to that line.
“Another Hello！”沒有成功，但沒關係。程式在到達該行之前崩潰。

Checking the exit code of the program:
檢查程式的退出代碼：

$ ./hello
Hello!
Segmentation fault (core dumped)
$ echo $?
139

fflush()  fflush（）

The fflush function also flushes the output buffer to the output device without the use of newline character (\n).
fflush 函數還會將輸出緩衝區刷新到輸出設備，而不使用換行符 （\n）。

#include <stdio.h>

int main(void)
{
        printf("%s", "Hello!");
        fflush(stdout);
        int *p = NULL;
        *p = 5;
        printf("%s\n", "Another Hello!");
}

will output  將輸出

$ gcc -Wall -Wextra -o hello hello.c && ./hello
Hello!Segmentation fault (core dumped)

stderror is always unbuffered
stdError 始終是 unbuffered

Someone on HN said
HN 上有人說

If you use stderr, you get this for free: stderr is always unbuffered (see here).
如果你使用 stderr，你可以免費獲得這個： stderr 總是無緩衝的（見這裡）。

#include <stdio.h>

int main(void)
{
        fprintf(stderr, "%s", "Hello!");
        int *p = NULL;
        *p = 5;
        printf("%s\n", "Another Hello!");
}

works the same  工作原理相同

$ gcc -Wall -Wextra -o hello_stderr hello_stderr.c && ./hello_stderr
Hello!Segmentation fault (core dumped)

Same exit code as the previous example as well.
退出代碼也與前面的示例相同。

$ echo $?
139

嗯，這就是我的小 TIL，希望你也能學到一些東西！
