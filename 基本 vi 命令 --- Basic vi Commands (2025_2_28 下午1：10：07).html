Basic vi Commands   基本 vi 命令

What is vi?  什麼是 vi ？

The default editor that comes with the UNIX operating system is called vi (visual editor). [Alternate editors for UNIX environments include pico and emacs, a product of GNU.]
UNIX作系統附帶的默認編輯器稱為 vi （visual editor）。[UNIX 環境的備用編輯器包括 pico 和 emacs ，GNU 的產物。

The UNIX vi editor is a full screen editor and has two modes of operation:
UNIX vi 編輯器是一個全螢幕編輯器，具有兩種模式 操作：
  1. Command mode commands which cause action to be taken on the file, and
    命令模式命令，這些命令會導致對檔案執行作，以及
  2. Insert mode in which entered text is inserted into the file.
    插入模式，在該模式下，輸入的文字將插入到檔中。
In the command mode, every character typed is a command that does something to the text file being edited; a character typed in the command mode may even cause the vi editor to enter the insert mode. In the insert mode, every character typed is added to the text in the file; pressing the <Esc> (Escape) key turns off the Insert mode.
在命令模式下，鍵入的每個字元都是一個命令，該命令 對正在編輯的文字檔案進行一些作; 在 命令模式甚至可能導致 vi 編輯器進入 插入模式。 在插入模式下，鍵入的每個字元都會加入到文字中 在檔中; 按 <Esc> （Escape） 鍵 關閉 Insert 模式。

While there are a number of vi commands, just a handful of these is usually sufficient for beginning vi users. To assist such users, this Web page contains a sampling of basic vi commands. The most basic and useful commands are marked with an asterisk (* or star) in the tables below. With practice, these commands should become automatic.
雖然有許多 vi 命令，但對於初學者 vi 來說，通常只有少數幾個命令就足夠了。為了幫助這些使用者，此 Web 頁包含了一些基本 vi 命令。在下表中，最基本和最有用的命令標有星號（ * 或星號）。通過練習，這些命令應該會變得自動。

NOTE: Both UNIX and vi are case-sensitive. Be sure not to use a capital letter in place of a lowercase letter; the results will not be what you expect.
注意：UNIX 和 vi 都區分大小寫。一定不要用大寫字母代替小寫字母;結果不會是你所期望的。

To Get Into and Out Of vi
進出 vi

To Start vi  開始 vi

To use vi on a file, type in vi filename. If the file named filename exists, then the first page (or screen) of the file will be displayed; if the file does not exist, then an empty file and screen are created into which you may enter text.
在檔案上使用 vi ，請鍵入 vi filename 。如果 name filename 的檔案存在，則將顯示檔的第一頁（或螢幕）;如果檔案不存在，則會創建一個空檔和螢幕，您可以在其中輸入文字。

* vi filename edit filename starting at line 1
從第 1 行開始編輯 filename
  vi -r filename recover filename that was being edited when system crashed
恢復 filename 系統崩潰時正在編輯的內容

To Exit vi  退出 vi

Usually the new or modified file is saved when you leave vi. However, it is also possible to quit vi without saving the file.
通常，當您離開 vi 時，會保存新的或修改的檔。 但是，也可以退出 vi 而不保存檔。

Note: The cursor moves to bottom of screen whenever a colon (:) is typed. This type of command is completed by hitting the <Return> (or <Enter>) key.
注意：每當鍵入冒號 （ : ） 時，游標都會移動到螢幕底部。這種類型的命令是通過點擊 <Return> （或 <Enter> ） 鍵來完成的。

* :x<Return> quit vi, writing out modified file to file named in original invocation
quit vi ，將修改後的檔寫出到原始調用中命名的檔
  :wq<Return> quit vi, writing out modified file to file named in original invocation
quit vi ，將修改後的檔寫出到原始調用中命名的檔
  :q<Return> quit (or exit) vi  離開 （或離開） vi
* :q!<Return> quit vi even though latest changes have not been saved for this vi call
即使尚未為此 vi 調用保存最新更改， vi 也要退出

Moving the Cursor  移動游標

Unlike many of the PC and MacIntosh editors, the mouse does not move the cursor within the vi editor screen (or window). You must use the the key commands listed below. On some UNIX platforms, the arrow keys may be used as well; however, since vi was designed with the Qwerty keyboard (containing no arrow keys) in mind, the arrow keys sometimes produce strange effects in vi and should be avoided.
與許多 PC 和 MacIntosh 編輯器不同，滑鼠不會在 vi 編輯器螢幕（或視窗）內移動游標。您必須使用下面列出的鍵盤命令。在某些 UNIX 平臺上，也可以使用箭頭鍵;但是，由於在設計時考慮了 Qwerty 鍵盤（不包含方向鍵），因此 vi 方向鍵有時會產生奇怪的效果 vi ，因此應避免使用。

If you go back and forth between a PC environment and a UNIX environment, you may find that this dissimilarity in methods for cursor movement is the most frustrating difference between the two.
如果您在 PC 環境和 UNIX 環境之間來回切換，您可能會發現游標移動方法的這種差異是兩者之間最令人沮喪的差異。

In the table below, the symbol ^ before a letter means that the <Ctrl> key should be held down while the letter key is pressed.
在下表中，字母前的符號 ^ 表示在按下字母鍵時應按住該 <Ctrl> 鍵。

* j or <Return>
j 或 <Return>
  [or down-arrow]   [或向下箭頭]		 move cursor down one line
將游標向下移動一行
* k [or up-arrow]
k [或向上箭頭]		 move cursor up one line
將游標上移一行
* h or <Backspace>
h 或 <Backspace>
  [or left-arrow]   [或向左箭頭]		 move cursor left one character
將游標向左移動一個字元
* l or <Space>
l 或 <Space>
  [or right-arrow]
[或向右箭頭]		 move cursor right one character
將游標向右移動 1 個字元
* 0 (zero)    0 （零） move cursor to start of current line (the one with the cursor)
將游標移至目前行的開頭（帶有游標的行）
* $ move cursor to end of current line
將游標移至當前行的末尾
  w move cursor to beginning of next word
將游標移動到下一個單詞的開頭
  b move cursor back to beginning of preceding word
將游標移回前一個單詞的開頭
  :0<Return> or 1G move cursor to first line in file
將游標移至檔案中的第一行
  :n<Return> or nG move cursor to line n
將游標移動到行 n
  :$<Return> or G move cursor to last line in file
將游標移至檔案中的最後一行

Screen Manipulation  螢幕作

The following commands allow the vi editor screen (or window) to move up or down several lines and to be refreshed.
以下命令允許 vi 編輯器螢幕（或視窗）向上或向下移動幾行並刷新。

  ^f move forward one screen  前進一個螢幕
  ^b move backward one screen  向後移動一個螢幕
  ^d move down (forward) one half screen
向下（向前）移動半個螢幕
  ^u move up (back) one half screen
上移（後移）半個螢幕
  ^l redraws the screen  重繪螢幕
  ^r redraws the screen, removing deleted lines
重繪螢幕，刪除已刪除的行

Adding, Changing, and Deleting Text
加入、更改和刪除文字

Unlike PC editors, you cannot replace or delete text by highlighting it with the mouse. Instead use the commands in the following tables.
與 PC 編輯器不同，您不能通過用滑鼠突出顯示文字來替換或刪除文字。請改用下表中的命令。

Perhaps the most important command is the one that allows you to back up and undo your last action. Unfortunately, this command acts like a toggle, undoing and redoing your most recent action. You cannot go back more than one step.
也許最重要的命令是允許您備份和撤銷上一個作的命令。不幸的是，此命令的作用類似於切換、撤銷和重做您最近的作。您不能返回多個步驟。

* u UNDO WHATEVER YOU JUST DID; a simple toggle
撤消您剛剛所做的任何事情;一個簡單的切換

The main purpose of an editor is to create, add, or modify text for a file.
編輯器的主要用途是創建、加入或修改檔的文字。

Inserting or Adding Text  插入或加入文字

The following commands allow you to insert and add text. Each of these commands puts the vi editor into insert mode; thus, the <Esc> key must be pressed to terminate the entry of text and to put the vi editor back into command mode.
以下命令允許您插入和加入文字。這些命令中的每一個都將 vi 編輯器置於插入模式;因此，必須按下該 <Esc> 鍵才能終止文字的輸入並將 vi 編輯器重新置於命令模式。

* i insert text before cursor, until <Esc> hit
在游標之前插入文字，直到 <Esc> 命中
  I insert text at beginning of current line, until <Esc> hit
在當前行的開頭插入文字，直到 <Esc> 命中
* a append text after cursor, until <Esc> hit
在游標後附加文字，直到 <Esc> 命中
  A append text to end of current line, until <Esc> hit
將文字附加到當前行的末尾，直到 <Esc> 命中
* o open and put text in a new line below current line, until <Esc> hit
打開並將文字放在當前行下方的新行中，直到 <Esc> 點擊
* O open and put text in a new line above current line, until <Esc> hit
打開並將文字放在當前行上方的新行中，直到 <Esc> 點擊

Changing Text  更改文字

The following commands allow you to modify text.
以下命令允許您修改文字。

* r replace single character under cursor (no <Esc> needed)
取代游標下的單個字元（不需要 <Esc> ）
  R replace characters, starting with current cursor position, until <Esc> hit
替換字元，從當前游標位置開始，直到 <Esc> 命中
  cw change the current word with new text,
用新文字更改當前單詞，
starting with the character under cursor, until <Esc> hit
從 游標下的字元，直到 <Esc> 點擊
  cNw change N words beginning with character under cursor, until <Esc> hit;
更改 N 以游標下的字元開頭的單詞， 直到 <Esc> 被擊中;
  e.g., c5w changes 5 words
例： c5w 更改 5 個單詞
  C change (replace) the characters in the current line, until <Esc> hit
更改（替換）當前行中的字元，直到 <Esc> 命中
  cc change (replace) the entire current line, stopping when <Esc> is hit
更改（替換）整個當前行，命中時 <Esc> 停止
  Ncc or cNc
Ncc 或 cNc 		change (replace) the next N lines, starting with the current line,
更改（替換）接下來的 N 行， 從當前行開始，
stopping when <Esc> is hit
被擊中時 <Esc> 停止

Deleting Text  刪除文字

The following commands allow you to delete text.
以下命令允許您刪除文字。

* x delete single character under cursor
刪除游標下的單個字元
  Nx delete N characters, starting with character under cursor
刪除 N 個字元，從游標下的字元開始
  dw delete the single word beginning with character under cursor
刪除游標下以字元開頭的單個單詞
  dNw delete N words beginning with character under cursor;
刪除 N 游標下以字元開頭的單詞;
  e.g., d5w deletes 5 words
例如， d5w 刪除5個單詞
  D delete the remainder of the line, starting with current cursor position
刪除該行的其餘部分，從當前游標位置開始
* dd delete entire current line
刪除整個當前行
  Ndd or dNd
Ndd 或 dNd 		delete N lines, beginning with the current line;
刪除 N 行，從當前行開始;
  e.g., 5dd deletes 5 lines
例如， 5dd 刪除 5 行

Cutting and Pasting Text  剪下和粘貼文字

The following commands allow you to copy and paste text.
以下命令允許您複製和貼上文字。

  yy copy (yank, cut) the current line into the buffer
將當前行複製 （rank， cut） 到緩衝區中
  Nyy or yNy
Nyy 或 yNy 		copy (yank, cut) the next N lines, including the current line, into the buffer
將接下來的 N 行（包括當前行）複製 （拉取、剪下） 到緩衝區中
  p put (paste) the line(s) in the buffer into the text after the current line
將緩衝區中的行 （粘貼） 到當前行之後的文字中

Other Commands  其他命令

Searching Text  搜尋文字

A common occurrence in text editing is to replace one word or phase by another. To locate instances of particular sets of characters (or strings), use the following commands.
文字編輯中的常見情況是將一個單詞或階段替換為另一個單詞或階段。要尋找特定字元集（或字串）的實例，請使用以下命令。

  /string search forward for occurrence of string in text
向前搜尋文字中出現的 string
  ?string search backward for occurrence of string in text
向後搜尋文字中出現的 string
  n move to next occurrence of search string
移動到搜尋字串的下一個匹配項
  N move to next occurrence of search string in opposite direction
以相反方向移動到下一個出現的搜尋字串

Determining Line Numbers  確定行號

Being able to determine the line number of the current line or the total number of lines in the file being edited is sometimes useful.
能夠確定當前行的行號或正在編輯的檔中的總行數有時很有用。

  :.= returns line number of current line at bottom of screen
返回螢幕底部當前行的行號
  := returns the total number of lines at bottom of screen
返回螢幕底部的總行數
  ^g provides the current line number, along with the total number of lines,
提供當前行號以及總計 行數 /
in the file at the bottom of the screen
在螢幕底部的檔中

Saving and Reading Files  保存和讀取檔

These commands permit you to input and output files other than the named file with which you are currently working.
這些命令允許您輸入和輸出目前正在使用的命名檔以外的檔案。

  :r filename<Return> read file named filename and insert after current line
讀取檔名 filename 並在其後插入 當前線路
(the line with cursor)  （帶游標的行）
  :w<Return> write current contents to file named in original vi call
將當前內容寫入 Original vi 調用中命名的檔案
  :w newfile<Return> write current contents to a new file named newfile
將當前內容寫入名為 newfile
  :12,35w smallfile<Return> write the contents of the lines numbered 12 through 35 to a new file named smallfile
將編號為 12 到 35 的行的內容寫入名為 smallfile
  :w! prevfile<Return> write current contents over a pre-existing file named prevfile
將當前內容寫入預先存在的名為 prevfile
