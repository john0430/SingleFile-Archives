2024-10-29

今天發現我提給 VS Code 的 PR 被官方感謝了。深受鼓舞！

這個 PR 做了什麼

這個 PR 在 vscode-languageserver-node 倉庫下。它是一個 npm 項目，為 Node 應用提供了語言伺服器封裝，用於通過 Node 開發語言伺服器應用。

在語言伺服器協議（LSP）中，有一個 Semantic Tokens 功能，用來實現基於語義的程式碼高亮。
它是這樣工作的：程式碼編輯器向語言伺服器請求 Semantic Tokens，語言伺服器返回一個列表，裡邊是每個 token 的位置和語義類型。

在詞法分析中，MyClass 是一個 token，TypeScript 語言伺服器標記它的語義是 class，而 myProperty 的語義是 property。

程式碼編輯器根據語義類型，給這個位置的 token 上色。例如 class 是紅色，property 是藍色。

vscode-languageserver-node 封裝了程式碼編輯器（客戶端）和語言伺服器基於 LSP 通訊的細節。它在收集語義列表時有個 bug：

語言伺服器可能會提供和文件字元流順序不一樣的 token 列表。
例如 myProperty 在前，MyClass 在後。翻閱文件後我發現，LSP 並沒有約束 token 的順序一定要和字元流一致，亂序是可接受的。
但在實際表現中，使用這個包的語言伺服器會過濾亂序的 token，剩下有序的 token 返回給客戶端。

我建了一個倉庫 來驗證 VS Code API 支援亂序 token，而使用這個包的語言伺服器不支援

我的 PR 對此做了修復，微軟的大佬回應也很快很積極。幾番 review 後得到了合併。

它如何影響 VS Code

VS Code 通過外掛機制，橫向擴展它支援的程式語言。想在 VS Code 裡編寫你想要的語言，只需要下載對應的語言外掛即可。

VS Code 內建了 TypeScript 外掛，它基於 LSP 為 VS Code 內的 .ts 檔案提供語言服務。
它的語言伺服器（被稱為 tsserver）就是 Node 應用，它使用 vscode-languageserver-node 這個包做封裝。

也就是說在 VS Code 內寫 TypeScript，就會用到這個包。很開心，這意味著 2024 年 5 月起 VS Code 使用者會使用幾行我寫的程式碼 :)

語義和語法高亮

在 LSP 中定義了基於語義的高亮，也就是語言伺服器編譯程式碼，分析每個 token 具體的語義資訊，告知客戶端。

VS Code 還提供了一種更輕量的，基於語法的高亮。它無需編譯，而是通過正則匹配來高亮。
